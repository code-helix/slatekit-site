Install

All software below is required to run Kotlin and Slate Kit. Kotlin is dependent on Java. Gradle is used to build and package Kotlin projects and is the build tool for Slate Kit.

Required

Tech Version About Link
Java 1.8 Java see below
Kotlin 1.1.2 Kotlin see below


Suggested

The software below is recommended but not required. Slate Kit supports MySql with support for PostGres and Sql Server coming later. Having the MySql Java Connector will help with the sample apps and is also needed for the ORM ( entities ).

Tech Version About Link
IntelliJ 2017.2 IDE download
Gradle 3.5 Build tool see below


Optional

Slate Kit supports building Web APIs using Spark Java, Cloud Services ( Files, Queues ) using AWS and databases using MySql. The following are needed if you plan on using any of these.

Tech Version About Link
MySql 5.7 Database Server download
MySql Connector 5.7 For database connectivity and orm download
Spark Java 2.6 For Slate API Server visit
AWS Sdk 1.10.55 File Storage( S3 ), Queues( SQS ) download

Setup

You can reference all the Slate Kit modules in gradle. The binaries are available from bintray. First you just have to include the Slate Kit maven url. Only the results, utilities modules are listed here for sample purposes. You can include additional projects 

    // other setup ...
    repositories {
        maven { url  "https://dl.bintray.com/codehelixinc/slatekit" }
    }

    dependencies {
        // other libraries
        
        // slatekit-results: Result<T,E> to model successes/failures with optional status codes
        compile 'com.slatekit:slatekit-results:0.9.9'
        
        // slatekit-common: Utilities for Android or Server
        compile 'com.slatekit:slatekit-common:0.9.9'
        
        // Misc architecture components ( all depend on results/common components above )
        // Use only what you need ( see artifacts next section )

    }

Available

All the components available are located in various projects for the purpose of organization and modularity. Take a look at the projects to get an understanding of the purpose of each project and the dependencies between them. You can use as little of Slate Kit or as much as you need. The simplest way to start is with the Results component, followed by the Utilities. Then you can proceed to checking out the concepts, sample apps and using just the individual libraries that you need.

Component Docs Source Description
Results slatekit-results Models successes and failures with optional status codes
Common slatekit-common General purpose utility components to extend the standard library
App slatekit-app An app template with support for args, envs, logs, configs, help and more
API slatekit-apis Components for building Univeral APIS that can run on the CLI and Web
CLI slatekit-cli A base template to build command line interaces for accessing your components
Cloud slatekit-cloud A base template to build command line interaces for accessing your components
Jobs slatekit-jobs A base template to build command line interaces for accessing your components
Entities slatekit-entities Light-weight ORM with a DDD/Entity Services approach
Server slatekit-server General purpose utility components to extend the standard library
